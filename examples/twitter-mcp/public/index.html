twitter-mcp
Hosted MCP Server · Twitter / X · OAuth 2.0 + PKCE

Your timeline
as LLM tools.

twitter-mcp is a multi-user MCP server that exposes Twitter/X — posts, timeline, search, engagement, lists, bookmarks — as tools any MCP-compatible LLM client can call. Confidential-client OAuth 2.0 with PKCE; per-user tokens stored in Neon Postgres. Built on the umwelten/mcp-serve library.

Live Endpoint → Source on GitHub umwelten docs

01 · Connect

Add to your MCP client

Pick the client you use. OAuth runs automatically on first connection — your browser opens, you authorize Twitter, and you're in.

03 · Under the hood

OAuth 2.0 + PKCE, chained

Why PKCE matters here

Twitter rejects authorize requests without a code_challenge — even for confidential clients. TwitterProvider generates a verifier per flow, keys it by state, and consumes it at exchange via the upstreamState parameter of mcp-serve's UpstreamOAuthProvider.exchangeCode.

01

Client hits /mcp

The server returns 401 with WWW-Authenticate. Client walks /.well-known/oauth-authorization-server to discover endpoints and registers dynamically at POST /oauth/register.

02

MCP-side PKCE

Client generates its own code verifier + challenge and redirects the user to /oauth/authorize.

03

Twitter-side PKCE (the gotcha)

Server generates a second verifier, stores it keyed by state, and redirects the user to Twitter with its challenge. Without this step Twitter returns error=invalid_request.

04

Token exchange

Twitter redirects back; server exchanges the code (with the saved verifier + Basic auth) for Twitter access/refresh tokens. Tokens are stored per-user in Neon Postgres.

05

MCP token issued

Server mints an MCP auth code and redirects back to the client. Client exchanges it at POST /oauth/token. Authenticated /mcp requests carry Authorization: Bearer <mcp-token>.

04 · Run your own

Self-host or fork it

Source is small: a Twitter-specific UpstreamOAuthProvider (with PKCE), a Zod-validated tool registrar, a few lines of glue to mcp-serve. Fly.io + Neon free tier works for personal use.

01

Fork

examples/twitter-mcp/ in the umwelten repo.

02

Register a Twitter app

Type: Web App, Automated App or Bot. Permissions: Read (or Read and write). Callback: https://<your-host>/oauth/upstream-callback.

03

Provision Neon

Free tier is plenty. Run pnpm run db:setup once against DATABASE_URL.

04

Deploy

fly launch, set secrets, fly deploy. Docker build is self-contained.

05 · What this is built on

umwelten/mcp-serve

umwelten is Focus.AI's TypeScript toolkit for multi-provider LLM access, systematic evaluation, and agent environments. Its mcp-serve subpath gives you an HTTP MCP server with built-in OAuth 2.1, streamable transport, and a pluggable per-user token store — so wrapping a new API takes an UpstreamOAuthProvider and a tool registrar, not a custom auth stack. The upstreamState parameter supports upstreams like Twitter that require their own PKCE dance.

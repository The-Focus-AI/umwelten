oura-mcp
Hosted MCP Server · Oura Ring · OAuth 2.1

Your ring data
in any MCP client.

oura-mcp is a multi-user MCP server that exposes Oura Ring health data — sleep, readiness, activity, stress, heart rate — as tools any MCP-compatible LLM client can call. OAuth 2.1 with dynamic client registration; per-user tokens stored in Neon Postgres. Built on the umwelten/mcp-serve library.

Live Endpoint → Source on GitHub umwelten docs

01 · Connect

Add to your MCP client

Pick the client you use. OAuth runs automatically on first connection — your browser opens, you authorize with Oura, and you're done.

Claude Code

# registers the server for this project
claude mcp add --transport http oura https://focus-oura-mcp.fly.dev/mcp

umwelten CLI

# interactive chat with OAuth flow
dotenvx run -- pnpm run cli mcp chat \
  --url https://focus-oura-mcp.fly.dev/mcp

# one-shot question
dotenvx run -- pnpm run cli mcp chat \
  --url https://focus-oura-mcp.fly.dev/mcp \
  --one-shot "how did I sleep last week?"

Any MCP client (JSON config)

{
  "mcpServers": {
    "oura": {
      "type": "streamable-http",
      "url": "https://focus-oura-mcp.fly.dev/mcp"
    }
  }
}

02 · Available Tools

Six read-only tools

Each tool takes start_date (required, YYYY-MM-DD) and end_date (optional). Responses are raw Oura v2 API JSON.

ToolOura EndpointWhat it returns
oura_sleep/v2/usercollection/daily_sleepDaily sleep scores and contributors
oura_sleep_detail/v2/usercollection/sleepPer-sleep periods: deep/REM/light, lowest HR, avg HRV
oura_readiness/v2/usercollection/daily_readinessReadiness, temperature deviation, HRV balance
oura_activity/v2/usercollection/daily_activitySteps, calories, activity score
oura_stress/v2/usercollection/daily_stressStress/recovery minutes, day summary
oura_heart_rate/v2/usercollection/heartrateTime-series heart rate data

03 · Under the hood

How the OAuth chain works

01

Client hits /mcp

The server returns 401 with WWW-Authenticate pointing at the resource metadata document.

02

Discovery & registration

Client discovers OAuth endpoints from /.well-known/oauth-authorization-server and registers dynamically via POST /oauth/register.

03

Upstream chain to Oura

Your browser is sent to Oura to authorize. The server relays the callback and stores your Oura tokens per-user in Neon Postgres.

04

MCP token issued

Client exchanges the code at POST /oauth/token. Authenticated requests carry Authorization: Bearer <mcp-token>; the server resolves your user and uses your Oura token to call the upstream API.

04 · Run your own

Self-host or fork it

The source is small: a Twitter-flavored UpstreamOAuthProvider, a Zod-validated tool registrar, and a few lines to glue them to mcp-serve. Fly.io + Neon free tier works for personal use.

01

Fork

examples/oura-mcp/ in the umwelten repo — four small source files.

02

Register an Oura app

Callback URI: https://<your-fly-app>.fly.dev/oauth/upstream-callback. Scopes: daily, heartrate, personal.

03

Provision Neon

Free tier. Run pnpm run db:setup once against DATABASE_URL.

04

Deploy

fly launch, set secrets, fly deploy. Docker build is self-contained.

05 · What this is built on

umwelten/mcp-serve

umwelten is Focus.AI's TypeScript toolkit for multi-provider LLM access, systematic evaluation, and agent environments. Its mcp-serve subpath gives you an HTTP MCP server with built-in OAuth 2.1, streamable transport, and a pluggable per-user token store — so wrapping a new API takes an UpstreamOAuthProvider and a tool registrar, not a custom auth stack.

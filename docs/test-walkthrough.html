Focus.AI Labs · Exploration Browser Test Walkthrough

Is it actually delightful?

17 numbered tests you can run hands-on against the umwelten browse command. Each one maps to specific user stories from PRD #56 (Exploration Browser command-center redesign). Each one names the friction we should tighten.

Green callouts = expected delightful behavior. Yellow callouts = specific rough edges to fix. Run them in order; they build on each other.

Setup before you start

cd /Users/wschenk/The-Focus-AI/umwelten pnpm install # if you haven't already pnpm test:run # sanity check — should see 1211 tests pass

What you'll need

  • A project with at least a few AI coding sessions (pi / Claude Code / Cursor / Habitat)
  • Your .env populated with GOOGLE_GENERATIVE_AI_API_KEY (or another provider)
  • A real terminal (Ink needs raw stdin; tmux is fine; piped stdin is not)

Recommended target projects

  • ../artifacts.thefocus.ai — 29 pi sessions, all digested. Best for testing chat + filtering.
  • ./ (umwelten itself) — 79 sessions across all four sources, mostly undigested. Best for testing the full digest pipeline + the --force path.

Test 1 PRD #56 user stories US-1US-2

Launch and see something immediately

PASS

Single entrypoint must start instantly with cheap local metadata — no LLM call before the user has seen anything.

Steps

dotenvx run -- pnpm run cli -- browse -p ../artifacts.thefocus.ai
Expected behaviour
  • Dashboard renders within ~1s.
  • Header reads Explorations (29/29) · 30d (or similar — adjusts to your project).
  • Each row shows status · src · type · topic · msgs · tools · cand · started · last.
  • Bottom status bar shows the keymap.
  • No tokens consumed yet (no LLM calls in the discovery path).
🔍 Friction to tighten
  • No confirmation that this is a project with sessions before the TUI starts — empty projects flash a "No explorations" message and exit, which can read as a crash.
  • The default 30-day window may hide everything for a stale project. The "1/2/3/4" key bindings to widen the window aren't shown in the bottom hint.
  • No explicit indication of which adapters were initialized — silent if Cursor's SQLite isn't available, for instance.

Test 17 Combines US-1 → US-18 · the golden path

The golden path: planning session → ADR

PASS

If only one flow has to be delightful, it's this one. Five keystrokes from "what did I work on?" to "ADR is in the repo."

Steps

  1. 1
    umwelten browse -p . — see today's work at the top.
  2. 2
    Find an undigested planning-ish session. Press D.
  3. 3
    Wait ~45s. Watch the type column light up plan in yellow.
  4. 4
    Press R. Ask: "what decisions were made and what tradeoffs did we consider?"
  5. 5
    Read the answer. Press 2 to promote to ADR. Done.
  6. 6
    git diff docs/adr/ — see the new file.
What should feel delightful
  • You opened the project, you saw what you did, you asked a question, you got back a decision document. No setup. No copying summaries by hand.
  • The ADR is plain Markdown — diffable, hand-editable, ready to commit.
  • You can run this flow again tomorrow against a different session and the ADR file numbers auto-increment.
🔍 Friction that breaks the flow today
  • Cost discoverability: in step 2 the user has no idea what D will cost. A "≈$0.02, 45s ETA" preview would let them act with confidence.
  • Wait time: 45s is a long time to look at "compacting (6 beats)". Either fill the wait (preview the upcoming summary as it's built — already partially done) or set expectations up front.
  • Answer editability: in step 5 the user reads the answer and might want to tweak it before promoting. There's no "[e]dit and promote" option. Once promoted, they edit the file by hand.
  • ADR template: a real ADR has Status, Context, Decision, Consequences. We write title + body. Adding a template wouldn't slow anything down.
  • Lost session selection: after promoting, the dashboard re-renders but cursor returns to row 0. The session you just promoted is no longer highlighted.

What would make this delightful?

All "🔍 Friction" callouts above, grouped by where they hurt the golden path the most.

High impact · golden path

Cost preview before D / R / P — display "≈$0.0X, ~Ns" before any LLM call. Today every key feels like a leap of faith.

Editable reflection answer before promote — let the user massage the streamed answer before [2] writes the file. Today it's locked.

Preserve row selection across re-renders — after promote/digest, cursor returns to row 0. Should track the previously-highlighted row by id.

Cancel digest mid-flight — Esc should cancel the active LLM call, not exit the TUI.

Discoverability

Cost / ETA on the overlay — "29 explorations, ≈$0.85, ~22 min." Pure number wins.

"?" key opens CONTEXT.md glossary — terms like Exploration / Skill Candidate / ADR matter; the dashboard never explains them.

Rename the two "Beats" views — b = "Skim", detail tab stays "Beats". Eliminates the collision.

Filter pickers, not just text search — by source, by type, by date range. Less typing, more obvious.

Detail view (US-8 → US-12)

Surface all 5 PRD lanes — narrative, key learnings, fact candidates, skill candidates, open loops. Today only 3 are clearly visible.

Promote from inside the detail view — adding R/P keys here would save a navigate-out step.

JSONL candidate browser — make .umwelten/candidates/*.jsonl readable in the TUI without leaving for jq.

Chat (US-17 extension)

Save chat turn → reflection — currently you have to exit and use R.

Persist chats to .umwelten/chats/ — today they're lost on exit.

"Show context" command — the model sees the digest + beats + facts; the user can't.

Cancel streaming — once an answer starts, you wait it out.

ADRs (US-18)

MADR template — Status / Context / Decision / Consequences / Alternatives. Currently we write title + body.

Preview the filename + path before write — "[2] Promote to docs/adr/0043-2026-05-27-marker-managed-sections.md?"

Undo for accidental promote — at minimum, append "(unconfirmed)" suffix until a final confirmation.

Already delightful

The whole pipeline is plain Markdown. No database. Diff-friendly. Hand-editable. This is the foundation everything else builds on — don't lose it.

Live row updates during digest. Watching the topic transform mid-pipeline (firstPrompt → compaction summary → analyzer summary) and the type column light up is genuinely satisfying.

Force re-digest with overlay. Now works on fully-digested projects.

1211 unit tests pass. The pipeline has real coverage. Refactor with confidence.

Would you use this every day?

The PRD's structural goals are met. The mechanical pieces work: one entrypoint, projection from four sources, confirmation overlay, streaming extraction, classifier, promotion targets, project-local storage.

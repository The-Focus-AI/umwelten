Try it now

Three commands, end to end.

Run these against any project that has AI coding sessions on disk. The dashboard discovers and projects everything before any LLM call.

1. Open the command-center

dotenvx run -- pnpm run cli browse -p .
✓ Discovers sessions across pi / Claude Code / Cursor / Habitat, shows a top-down table, and gates extraction behind a confirmation overlay.

2. Confirm to extract

The overlay states how many Explorations need extraction, the model, the concurrency, and an explicit cost warning. Press y to launch — rows transition new → digesting → digested inline.

3. Drill into one

Arrow keys to navigate. Enter opens detail. v transcript. b beats. D re-digest a single row. / search. q quit.

What is happening underneath

initializeAdapters() registers Claude Code, pi, Cursor

projectSessions() discovers every source session

buildExploreBrowse() wraps each into an ExplorationBrowserEntry + digest

Dashboard mounts (React Ink)

ExtractionEngine.run() streams progress events

Digests + candidate JSONLs written under .umwelten/

#61 7f2ab6a

Digests live
with the project.

sessions project-local

New convention

Digests used to live under ~/.umwelten/digests/ (global, mixed across projects). Now they live next to the code:

<project>/.umwelten/digests/sessions/<encoded-id>.json

loadDigest() / saveDigest() handle the round-trip with URL-encoded session IDs so cross-platform paths stay safe.

Consequences

Knowledge travels with the project. Move the repo, the digests come along. Multi-project setups stop blending Explorations across boundaries.

#63 53a02c6

Extraction workflow engine.

core deep module

What it does

Coordinates LLM extraction across a list of Explorations. Detects scope (newest undigested first, plus stale), controls concurrency (default 1), streams progress, persists candidates project-locally.

packages/core/src/interaction/analysis/extraction-engine.ts

Progress event shape

{ explorationId: string, sessionId: string, phase: "pending" | "digesting" | "digested" | "failed", detail?: string, }

The dashboard buffers these events and flushes id-keyed row patches every 150ms to avoid tmux flicker.

Five lanes for
candidate knowledge.

core jsonl

Per-kind JSONL files

.umwelten/candidates/project-facts.jsonl

.umwelten/candidates/skill-candidates.jsonl

.umwelten/candidates/open-loops.jsonl

.umwelten/candidates/preferences.jsonl

.umwelten/candidates/mistakes.jsonl

Append-only, fault-tolerant

CandidatePersistence appends one JSON object per line per candidate. Reads gracefully skip malformed lines so a partial write never breaks the store. readAll() returns every kind keyed by name.

packages/core/src/interaction/knowledge/candidate-persistence.ts

Why JSONL

Crash-safe (each append is atomic), tail-friendly (you can tail -f a kind as the engine runs), easy to diff and grep. The promote step reads these and decides which become durable in AGENTS.md / FACTS.md / .agents/skills/.

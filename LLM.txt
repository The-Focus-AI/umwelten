# Umwelten — context for coding agents

TypeScript (Node 20+) framework for **Habitat** (agent workspaces), **model evaluation** (multi-model benchmarks with scoring), and **pairwise ranking** (Elo via LLM judge).

- Human docs: https://umwelten.thefocus.ai/
- Repo: https://github.com/The-Focus-AI/umwelten
- Maintainer-deep module map: **CLAUDE.md** in repo root

## Rules

- Package manager: **pnpm** only. Tests: **`pnpm test:run`** (not `pnpm test`, which watches).
- Local CLI: **`pnpm run cli`** (tsx → `src/cli/entry.ts`). With secrets: **`dotenvx run -- pnpm run cli -- …`**
- **Never use gemini-2 models** — use **gemini-3** (e.g. `gemini-3-flash-preview`).
- Model responses: read **`ModelResponse.content`** (string), **not** `.text`.

---

## Layer stack (bottom → top)

Each layer depends only on layers below.

1. **Foundation** — `src/costs/`, `src/rate-limit/`, `src/markdown/`, `src/schema/`, `src/mcp/` (includes `RemoteMcpClient` for OAuth connections to remote MCP servers)
2. **Providers** — `src/providers/` — `getModel(modelDetails)`, `validateModel()`, Google/OpenRouter/Ollama/LM Studio/LlamaBarn/llama-swap/GitHub/DeepInfra/Fireworks/MiniMax/Nvidia. `llamaswap-config.ts` + `umwelten models llamaswap-config` generate a llama-swap YAML from local GGUF caches.
3. **Core runtime** — `src/cognition/` (runners), `src/stimulus/` (prompt config), `src/interaction/` (conversation)
4. **Context** — `src/context/` — compaction, token estimates
5. **Memory** — `src/memory/` — fact extraction when `stimulus.runnerType === 'memory'`
6. **Habitat** — `src/habitat/` — `Habitat.create()`, tool sets, sessions, secrets, sub-agents, bridge
7. **Evaluation** — `src/evaluation/` — `EvalSuite`, `PairwiseRanker`, strategies, caching, Dagger, `combine/` for suite reports
8. **Surfaces** — `src/cli/`, `src/ui/` (Telegram, Discord, TUI, web)

---

## Core types

- **`Stimulus`** — `role`, `objective`, `instructions`, `tools`, `temperature`, `runnerType`, `skillsDirs`. Pure config; does not run the model.
- **`Interaction`** — Messages + model + stimulus; **`chat()`, `generateText()`, `streamText()`, `generateObject()`, `compactContext()`**. Has **`userId`** field forwarded to OpenRouter/Anthropic.
- **`Habitat`** — Binds work directory, registers tool sets, loads prompts, manages sessions and agents.

## Evaluation — EvalSuite (recommended)

`EvalSuite` (`src/evaluation/suite.ts`) is the primary way to create evaluations. Define tasks + stimulus + models → get scored results + leaderboard in ~60-100 lines.

### Minimal example

```typescript
import { EvalSuite } from '../../src/evaluation/suite.js';

const suite = new EvalSuite({
  name: 'my-eval',
  stimulus: { role: 'helpful assistant', temperature: 0.3, maxTokens: 500 },
  models: [
    { name: 'gemini-3-flash-preview', provider: 'google' },
    { name: 'openai/gpt-5.4-nano', provider: 'openrouter' },
  ],
  tasks: [{
    id: 'q1', prompt: 'What is 2+2?', maxScore: 1,
    verify: (r) => ({ score: r.trim() === '4' ? 1 : 0, details: r.trim() }),
  }],
});
await suite.run();
```

### Two scoring modes

**VerifyTask** — deterministic, no LLM:
```typescript
{ id: 'test', prompt: '...', maxScore: 5,
  verify: (response) => ({ score: 3, details: 'partial match' }) }
```

**JudgeTask** — LLM judge with Zod schema:
```typescript
{ id: 'test', prompt: '...', maxScore: 5,
  judge: {
    schema: z.object({
      reasoning_quality: z.coerce.number().min(1).max(5),
      explanation: z.string(),
    }),
    instructions: ['Score 5 for correct, 1 for wrong.'],
    extractScore: (r) => r.reasoning_quality,  // optional, defaults to r.reasoning_quality ?? r.score ?? 0
  } }
```

### Config

```typescript
interface EvalSuiteConfig {
  name: string;                    // Output directory name
  stimulus: StimulusOptions | ((task) => StimulusOptions);
  tasks: EvalTask[];               // VerifyTask[] | JudgeTask[]
  models?: ModelDetails[];         // Default model list
  allModels?: ModelDetails[];      // Used when --all flag is passed
  judgeModel?: ModelDetails;       // Default: claude-haiku-4.5 via openrouter
  concurrency?: number;            // Default: 5
  judgeDelayMs?: number;           // Default: 500
}
```

CLI flags: `--all` (full models), `--new` (fresh run), `--run N` (resume specific run).

### Caching

Two layers, both automatic:
1. **Response cache** — `SimpleEvaluation` caches raw LLM responses per model/task. Resume by re-running without `--new`.
2. **Result cache** — `EvalSuite` caches scored results (score + judge output). Error/zero-score judge results auto-retry.

Results are written to `output/evaluations/{name}/runs/{NNN}/{taskId}/{modelKey}.json` — compatible with `eval combine`.

## Evaluation — PairwiseRanker

`PairwiseRanker` (`src/evaluation/ranking/pairwise-ranker.ts`) ranks model responses head-to-head using an LLM judge and produces Elo ratings.

```typescript
import { PairwiseRanker } from '../../src/evaluation/ranking/index.js';

const ranker = new PairwiseRanker(entries, {
  judgeModel: { name: 'anthropic/claude-haiku-4.5', provider: 'openrouter' },
  judgeInstructions: ['Compare these two responses. Which is better?'],
  pairingMode: 'swiss',  // or 'all' for round-robin
  swissRounds: 5,
  cacheDir: './output/rankings/my-ranking',
});

const output = await ranker.rank();
// output.rankings: RankedModel[] sorted by Elo descending
```

Features: Swiss tournament or round-robin pairing, position bias mitigation (random A/B flip), comparison-level caching, Bradley-Terry Elo (K=32), error resilience (ties on failure).

Pure math functions also exported: `expectedScore()`, `updateElo()`, `buildStandings()`, `allPairs()`, `swissPairs()`.

## Evaluation — Lower-level strategies

| Strategy | Purpose | Constructor |
|----------|---------|-------------|
| `SimpleEvaluation` | 1 prompt × N models (used by EvalSuite internally) | `new SimpleEvaluation(stimulus, models, prompt, cache, config)` |
| `MatrixEvaluation` | Prompt with `{placeholders}` × cartesian product | `new MatrixEvaluation(stimulus, models, prompt, cache, { dimensions })` |
| `BatchEvaluation` | N content items × N models with template prompt | `new BatchEvaluation(stimulus, models, prompt, cache, { items })` |

## Evaluation — Suite combine

Run independent evaluations, then aggregate into a unified leaderboard:

```typescript
import type { EvalDimension } from '../../src/evaluation/combine/types.js';

export const MY_SUITE: EvalDimension[] = [
  { evalName: 'my-reasoning', label: 'Reasoning', maxScore: 20,
    extractScore: (r) => r.score ?? 0, hasResultsSubdir: true },
  { evalName: 'my-knowledge', label: 'Knowledge', maxScore: 30,
    extractScore: (r) => r.correct ? 1 : 0 },
];
```

```bash
dotenvx run -- pnpm run cli eval combine --config suite-config.ts --format narrative --output report.md
```

Report formats: `console`, `md`, `json`, `narrative` (full prose writeup).

## Habitat

Top-level agent system. `Habitat.create({ workDir })` → register tools → `createInteraction()`.

Tool sets (all registered by default): file ops, time, URL (wget/markify/parse_feed), agent CRUD, sessions, external interactions (Claude/Cursor), agent runner (clone/logs/ask), secrets, search (Tavily), self-modify.

## Common tasks

| Goal | Where to start |
|------|----------------|
| Create a model eval | `new EvalSuite({ name, stimulus, models, tasks })` → `suite.run()` |
| Rank responses head-to-head | `new PairwiseRanker(entries, config)` → `ranker.rank()` |
| Multi-dimension leaderboard | Define `EvalDimension[]` → `eval combine --config ...` |
| Managed agent (tools, sessions) | `Habitat.create({ workDir })` → `createInteraction()` |
| Chat with remote MCP server | `pnpm run cli -- mcp chat --url https://server/mcp` |
| REPL / Telegram / Discord / web | `pnpm run cli -- habitat` |
| One-shot multi-model compare | `pnpm run cli -- eval run --models "provider:model,..." --prompt "..."` |
